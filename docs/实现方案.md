# 多 App 法务页 / 使用条款页 / 支持页实现方案

## 目标

在当前 Astro 项目中实现一个轻量级静态网站，用于统一展示多个 App 的：

```text
隐私政策页
使用条款页
支持页
```

每个 App 通过 URL path 区分：

```text
/{app}/privacy
/{app}/terms
/{app}/support
```

示例：

```text
/maxxing-ai/privacy
/maxxing-ai/terms
/maxxing-ai/support
```

## 技术栈

使用：

```text
Astro
React
Tailwind CSS v4
shadcn/ui
TypeScript
Markdown
```

不要使用 Next.js、数据库、后台管理系统。

原因：

```text
Astro 更适合静态内容站，Markdown 支持好。
React 用于 shadcn/ui 组件。
Tailwind CSS 和 shadcn/ui 作为整体样式与组件体系。
```

## shadcn/ui 安装方式

按照官方 Astro 文档的 Existing Project 路径执行：

参考：

```text
https://ui.shadcn.com/docs/installation/astro
```

当前项目不是空项目，因此不重新 `create astro`，而是在现有项目中增量接入：

```bash
npx astro add react tailwind --yes
```

然后在 `tsconfig.json` 中配置 `@/*` alias：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

初始化 shadcn：

```bash
npx shadcn@latest init --base radix --preset nova --yes
```

添加组件：

```bash
npx shadcn@latest add button card accordion separator --yes
```

## 路由规则

```text
/{app}/privacy -> 渲染 src/content/apps/{app}/privacy.md
/{app}/terms   -> 渲染 src/content/apps/{app}/terms.md
/{app}/support -> 使用 src/config/apps/{app}.json + default.json 渲染支持页
```

`privacy` 和 `terms` 使用各自 Markdown 文件生成静态路径：

```text
存在 privacy.md 才生成 /{app}/privacy
存在 terms.md 才生成 /{app}/terms
```

`support` 使用 App 内容目录生成静态路径：

```text
只要 src/content/apps/{app}/ 下存在 privacy.md 或 terms.md，就生成 /{app}/support
```

未知 App 不会进入 `getStaticPaths()`，静态部署时自然返回 404。

## 配置规则

每个 App 可选一个 JSON 配置：

```text
src/config/apps/default.json
src/config/apps/{app}.json
```

规则：

```text
1. default.json 必须存在。
2. default.json 必须提供 name、companyName、supportEmail。
3. {app}.json 可选。
4. {app}.json 存在时，覆盖 default.json 中的同名字段。
5. slug 不写在 JSON 里，从 URL path 中读取。
6. JSON 会在构建时校验，邮箱和 URL 格式错误会导致构建失败。
```

配置类型：

```ts
export type AppConfig = {
  slug: string;
  name: string;
  companyName: string;
  supportEmail: string;
  privacyEmail?: string;
  appStoreUrl?: string;
  googlePlayUrl?: string;
  websiteUrl?: string;
  faqs?: {
    question: string;
    answer: string;
  }[];
};
```

## 页面设计

整体样式走 shadcn/ui：

```text
全局 token：src/styles/global.css
布局：src/layouts/LegalLayout.astro
支持页：src/components/SupportPage.tsx
UI 组件：src/components/ui/*
```

`LegalLayout.astro` 负责：

```text
HTML 文档结构
SEO 基础 meta
canonical URL
App 名称页头
Privacy / Terms / Support 互链
页脚 companyName / websiteUrl
```

`Privacy` 和 `Terms` 页面：

```text
保持静态 Markdown 渲染
使用 legal-prose class 做 shadcn token 风格的正文排版
不需要客户端 JS
```

`Support` 页面：

```text
使用 React + shadcn/ui
使用 Card / Button / Accordion / Separator
因为 Accordion 需要交互，SupportPage 使用 client:load
```

## 新增 App 流程

新增一个 App 时，添加：

```text
src/content/apps/{app}/privacy.md
src/content/apps/{app}/terms.md
```

可选添加：

```text
src/config/apps/{app}.json
```

示例：

```text
src/content/apps/another-app/privacy.md
src/content/apps/another-app/terms.md
src/config/apps/another-app.json
```

构建后会生成：

```text
/another-app/privacy
/another-app/terms
/another-app/support
```

如果没有 `another-app.json`，支持页使用 `default.json`。

## 交付要求

```text
1. 使用 Astro，不使用 Next.js。
2. 使用 React + Tailwind CSS + shadcn/ui。
3. 整体样式走 shadcn/ui token 和组件体系。
4. /{app}/privacy 渲染 src/content/apps/{app}/privacy.md。
5. /{app}/terms 渲染 src/content/apps/{app}/terms.md。
6. /{app}/support 使用 src/config/apps/{app}.json 配置。
7. 缺失 {app}.json 时 fallback 到 default.json。
8. default.json 缺失或格式错误时构建失败。
9. JSON 中不写 slug，slug 从 URL path 中获取。
10. App 专属 JSON 覆盖 default.json 中的字段。
11. privacy / terms 静态路径分别按对应 Markdown 文件生成。
12. 不存在的 app path 返回 404。
13. 页面适配移动端。
14. 构建结果为静态 HTML。
15. 新增 App 只需要新增 Markdown 和可选 JSON。
```
