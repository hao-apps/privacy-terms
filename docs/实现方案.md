# 多 App 法务页 / 使用条款页 / 支持页网站实现方案

## 目标

开发一个轻量级静态网站，用于统一展示多个 App 的：

```text
隐私政策页
使用条款页
支持页
```

每个 App 通过 URL path 区分。

示例 URL：

```text
/maxxing-ai/privacy
/maxxing-ai/terms
/maxxing-ai/support

/another-app/privacy
/another-app/terms
/another-app/support
```

路由规则：

```text
/:app/privacy   -> 渲染 src/content/apps/{app}/privacy.md
/:app/terms     -> 渲染 src/content/apps/{app}/terms.md
/:app/support   -> 根据 src/config/apps/{app}.json 展示支持页
```

如果找不到 `src/config/apps/{app}.json`，则 fallback 使用：

```text
src/config/apps/default.json
```

## 技术栈

使用：

```text
Astro
React
Tailwind CSS
shadcn/ui
TypeScript
Markdown
```

不要使用 Next.js。

原因：

```text
Next.js 对这个需求太重，编译慢。
Astro 更适合静态内容站，Markdown 支持好，也可以使用 React 和 shadcn/ui。
```

## 配置文件设计

配置文件不再使用 `apps.ts`，改成每个 App 一个 JSON。

```text
src/config/apps/default.json
src/config/apps/maxxing-ai.json
src/config/apps/another-app.json
```

### default.json

`src/config/apps/default.json`

```json
{
  "name": "App",
  "companyName": "Your Company Name",
  "supportEmail": "support@example.com",
  "privacyEmail": "privacy@example.com",
  "websiteUrl": "https://example.com",
  "faqs": [
    {
      "question": "How do I cancel my subscription?",
      "answer": "Subscriptions are managed through your Apple ID subscription settings."
    },
    {
      "question": "How do I contact support?",
      "answer": "Email us with your app name, device model, OS version, and a description of the issue."
    }
  ]
}
```

### App 专属配置

`src/config/apps/maxxing-ai.json`

```json
{
  "name": "Maxxing AI",
  "companyName": "Your Company Name",
  "supportEmail": "maxxing-ai-support@example.com",
  "privacyEmail": "privacy@example.com",
  "appStoreUrl": "https://apps.apple.com/app/xxx",
  "websiteUrl": "https://example.com",
  "faqs": [
    {
      "question": "How do I cancel my Maxxing AI subscription?",
      "answer": "You can manage or cancel your subscription from your Apple ID subscription settings."
    },
    {
      "question": "Why is AI output sometimes inaccurate?",
      "answer": "AI-generated content may contain mistakes. Please verify important information."
    }
  ]
}
```

说明：

```text
1. default.json 必须存在。
2. {app}.json 是可选的。
3. 如果 {app}.json 存在，用它覆盖 default.json 中的同名字段。
4. 如果 {app}.json 不存在，完整使用 default.json。
5. slug 不写在 JSON 里，从 URL path 中读取。
```

## 配置加载器

创建：

```text
src/config/load-app-config.ts
```

```ts
import type { AppConfig } from "./types";

type RawAppConfig = Omit<AppConfig, "slug">;

const configModules = import.meta.glob("./apps/*.json", {
  eager: true,
  import: "default",
}) as Record<string, RawAppConfig>;

const defaultConfig = configModules["./apps/default.json"];

if (!defaultConfig) {
  throw new Error("Missing required config file: src/config/apps/default.json");
}

export function loadAppConfig(appSlug: string): AppConfig {
  const appConfig = configModules[`./apps/${appSlug}.json`];

  return {
    ...defaultConfig,
    ...appConfig,
    slug: appSlug,
  };
}
```

## App slug 来源

因为这是静态站，不能支持无限未知 path。

静态路径从 Markdown 目录中扫描：

```text
src/content/apps/{app}/privacy.md
src/content/apps/{app}/terms.md
```

也就是说，只要存在：

```text
src/content/apps/maxxing-ai/privacy.md
src/content/apps/maxxing-ai/terms.md
```

就生成：

```text
/maxxing-ai/privacy
/maxxing-ai/terms
/maxxing-ai/support
```

即使没有：

```text
src/config/apps/maxxing-ai.json
```

也会使用：

```text
src/config/apps/default.json
```

## app slug 扫描工具

创建：

```text
src/config/app-slugs.ts
```

```ts
const privacyModules = import.meta.glob("../content/apps/*/privacy.md");
const termsModules = import.meta.glob("../content/apps/*/terms.md");

function extractSlug(filePath: string): string | null {
  const match = filePath.match(/\/content\/apps\/([^/]+)\//);
  return match?.[1] ?? null;
}

export function getAppSlugs(): string[] {
  const slugs = new Set<string>();

  for (const filePath of Object.keys(privacyModules)) {
    const slug = extractSlug(filePath);
    if (slug) {
      slugs.add(slug);
    }
  }

  for (const filePath of Object.keys(termsModules)) {
    const slug = extractSlug(filePath);
    if (slug) {
      slugs.add(slug);
    }
  }

  return Array.from(slugs).sort();
}
```

## Privacy 页面

创建：

```text
src/pages/[app]/privacy.astro
```

```astro
---
import MarkdownLayout from "../../components/MarkdownLayout.astro";
import { loadAppConfig } from "../../config/load-app-config";
import { getAppSlugs } from "../../config/app-slugs";

export function getStaticPaths() {
  return getAppSlugs().map((app) => ({
    params: { app },
  }));
}

const { app: appSlug } = Astro.params;

if (!appSlug) {
  return Astro.redirect("/404");
}

const app = loadAppConfig(appSlug);

const modules = import.meta.glob("../../content/apps/*/privacy.md");

const key = `../../content/apps/${appSlug}/privacy.md`;
const loader = modules[key];

if (!loader) {
  return Astro.redirect("/404");
}

const mod: any = await loader();
const Content = mod.Content;
---

<MarkdownLayout app={app} title={`${app.name} Privacy Policy`}>
  <Content />
</MarkdownLayout>
```

## Support 页面组件

使用 shadcn/ui 开发支持页。

创建：

```text
src/components/SupportPage.tsx
```

```tsx
import type { AppConfig } from "@/config/types";
import { Button } from "@/components/ui/button";
import {
  Card,
  CardContent,
  CardHeader,
  CardTitle,
} from "@/components/ui/card";
import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from "@/components/ui/accordion";

type Props = {
  app: AppConfig;
};

export function SupportPage({ app }: Props) {
  return (
    <main className="mx-auto max-w-3xl px-6 py-10">
      <div className="mb-8">
        <p className="text-sm text-muted-foreground">{app.companyName}</p>
        <h1 className="mt-2 text-3xl font-bold">{app.name} Support</h1>
        <p className="mt-3 text-muted-foreground">
          Need help with {app.name}? Contact us below.
        </p>
      </div>

      <Card>
        <CardHeader>
          <CardTitle>Contact Support</CardTitle>
        </CardHeader>

        <CardContent className="space-y-4">
          <p className="text-sm text-muted-foreground">
            Please include your app name, device model, OS version, and a short
            description of the issue.
          </p>

          <Button asChild>
            <a href={`mailto:${app.supportEmail}`}>Email Support</a>
          </Button>
        </CardContent>
      </Card>

      <Card className="mt-6">
        <CardHeader>
          <CardTitle>Subscription Support</CardTitle>
        </CardHeader>

        <CardContent className="text-sm text-muted-foreground">
          If you purchased a subscription through the App Store, you can manage
          or cancel it from your Apple ID subscription settings.
        </CardContent>
      </Card>

      {app.faqs && app.faqs.length > 0 && (
        <Card className="mt-6">
          <CardHeader>
            <CardTitle>FAQ</CardTitle>
          </CardHeader>

          <CardContent>
            <Accordion type="single" collapsible>
              {app.faqs.map((faq, index) => (
                <AccordionItem key={index} value={`item-${index}`}>
                  <AccordionTrigger>{faq.question}</AccordionTrigger>
                  <AccordionContent>{faq.answer}</AccordionContent>
                </AccordionItem>
              ))}
            </Accordion>
          </CardContent>
        </Card>
      )}

      {app.appStoreUrl && (
        <div className="mt-6">
          <Button variant="outline" asChild>
            <a href={app.appStoreUrl}>View on the App Store</a>
          </Button>
        </div>
      )}

      {app.googlePlayUrl && (
        <div className="mt-3">
          <Button variant="outline" asChild>
            <a href={app.googlePlayUrl}>View on Google Play</a>
          </Button>
        </div>
      )}
    </main>
  );
}
```

## 交付要求

完成后需要满足：

```text
1. 使用 Astro，不使用 Next.js。
2. 使用 React + Tailwind CSS + shadcn/ui。
3. /{app}/privacy 渲染 src/content/apps/{app}/privacy.md。
4. /{app}/terms 渲染 src/content/apps/{app}/terms.md。
5. /{app}/support 使用 src/config/apps/{app}.json 配置。
6. 如果 src/config/apps/{app}.json 不存在，则 fallback 到 src/config/apps/default.json。
7. default.json 必须存在，否则构建失败。
8. JSON 中不需要写 slug，slug 从 URL path 中获取。
9. App 专属 JSON 可以覆盖 default.json 中的字段。
10. 静态路径根据 src/content/apps/{app}/ 下的 Markdown 文件生成。
11. 缺失 privacy.md 时，对应 privacy 页面返回 404。
12. 缺失 terms.md 时，对应 terms 页面返回 404。
13. 不存在的 app path 返回 404。
14. 页面适配移动端。
15. 构建结果应尽量是静态 HTML。
16. 支持页使用 shadcn/ui。
17. 新增 App 时，只需要新增 content/apps/{app}/privacy.md、terms.md，以及可选的 config/apps/{app}.json。
```

## 重要原则

```text
不要做数据库。
不要做后台管理。
不要用 Next.js。
不要把所有 App 配置集中写在 apps.ts。
不要把支持页内容完全写死在组件里。
App 差异信息放 src/config/apps/{app}.json。
默认配置放 src/config/apps/default.json。
隐私政策和使用条款放 Markdown。
构建结果应尽量是静态站点，方便部署到 Cloudflare Pages、Vercel、Netlify 或 nginx。
```

## 关键注意点

`default.json` 的作用是：

```text
某个已知 app 没有专属 JSON 配置时，使用默认配置。
```

它不是让任意未知 URL 都自动存在。

例如：

```text
/unknown-app/support
```

如果 `src/content/apps/unknown-app/` 不存在，则不应该生成页面，应该返回 404。

静态站必须提前知道有哪些 app，因此 app 列表来自：

```text
src/content/apps/{app}/
```
